struts2出现Unable to load configuration. - bean - jar:file:/…struts-default.xml问题

Idea的话 把out目录删除了再试, 意思就是把当前部署目录删除后重新编译生成

Tomcat java.lang.Exception: Socket bind failed: [730013]

端口被占用,需要换端口

org.apache.jasper.JasperException: Unable to compile class for JSP:

  • 我遇到的情况是, 使用了tomcat7.0+, 换成tomcat6.0+即可

  • 或者把javax.servlet.jar 和 javax.servlet.jsp.jar ,catalina.jar、jsp-api.jar、servlet-api.jar等这样的重复包，把它们从工程中删了就行（对于上面的异常，原因应该是包jsp-api.jar）
